# FP&A Performance Dashboard (Flagship Project)

FP&A performance dashboard focused on automated reporting, KPI analysis, and forecasting to support management control and decision-making.


## Business Context
Management control and FP&A teams need reliable, automated performance reporting to monitor financial results, identify trends, and support planning and decision-making.

This project simulates an FP&A use case in a retail / business context, focusing on performance analysis and reporting automation.

## **Objective**
- Automate recurring FP&A and management control reporting
- Analyze financial and performance indicators to identify trends and variances
- Support management decision-making through clear and actionable dashboards

## Data
The dataset represents historical sales and performance data with:
- Revenue
- Costs
- Volume
- Time dimension (monthly)

Data is used for analytical and educational purposes.

## Methodology
- SQL for data extraction and KPI calculation
- Excel / Power BI for data modeling and visualization
- Simple forecasting techniques for planning support

## Key KPIs
- Revenue
- Cost
- Margin (%)
- Volume trends
- Month-over-Month and Year-over-Year evolution
- Forecast vs Actual

## Insights & Recommendations
- Identification of performance trends and seasonality
- Analysis of margin evolution over time
- Recommendations to improve reporting efficiency and decision visibility

## Next Improvements
- Integration of inventory and stock-related indicators
- More advanced forecasting models
- Automation with scheduled data refresh
## Tools Used
- SQL (data extraction and KPI calculation)
- Excel (analysis and calculations)
- Power BI (data modeling and visualization)
## **SQL Analysis**
Basic SQL queries are used to calculate FP&A and management control KPIs from the raw retail dataset.

- Total revenue calculation
- Margin analysis (revenue, profit, margin %)
